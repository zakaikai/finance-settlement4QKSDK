# 模板配置词条

本文档说明如何配置对账单 Excel 模板，将数据词条（字段）对应到模板的指定区域位置。

## 概述

对账单模板是一个普通的 `.xlsx` 文件，在单元格中使用 `{{标记}}` 占位符来标记数据位置。系统渲染时会替换这些标记为实际数据。

模板文件上传到"系统管理 → 对账模板"后，在结算页面导出时选择即可使用。

## 标记类型

### 1. 标量标记 — 单个值替换

直接写在单元格中，渲染时替换为对应值。

| 标记 | 说明 | 示例值 |
|------|------|--------|
| `{{title}}` | 标题，根据结算类型自动生成 | "收入结算对账单" / "付款结算对账单" |
| `{{bill_no}}` | 对账单编号 | ST-20260509-1430 |
| `{{period}}` | 对账周期 | "2026-03" 或 "2026-01 ~ 2026-03" |
| `{{date}}` | 生成日期 | "2026年05月09日" |
| `{{notes}}` | 备注文本 | 默认对账说明 |
| `{{party_a.name}}` | 甲方名称 | |
| `{{party_a.address}}` | 甲方地址 | |
| `{{party_a.phone}}` | 甲方电话 | |
| `{{party_a.bank_name}}` | 甲方开户银行 | |
| `{{party_a.bank_account}}` | 甲方银行账号 | |
| `{{party_a.tax_id}}` | 甲方税号 | |
| `{{party_b.name}}` | 乙方名称 | |
| `{{party_b.address}}` | 乙方地址 | |
| `{{party_b.phone}}` | 乙方电话 | |
| `{{party_b.bank_name}}` | 乙方开户银行 | |
| `{{party_b.bank_account}}` | 乙方银行账号 | |
| `{{party_b.tax_id}}` | 乙方税号 | |

**用法示例：**

```
对账单编号: {{bill_no}}
对账周期: {{period}}
甲方: {{party_a.name}}
开户行: {{party_a.bank_name}}      账号: {{party_a.bank_account}}
```

### 2. 表格标记 — 数据行区域

表格区域用开始/结束标记包裹，系统会自动展开多行数据。

| 标记 | 说明 |
|------|------|
| `{{#table:rows}}` | 表格区域开始（放在区域第一行） |
| `{{/table}}` | 表格区域结束（放在区域最后一行） |
| `{{row.字段名}}` | 行内数据字段（只放在表格区域内） |
| `{{row.seq}}` | 行序号（自动生成 1, 2, 3...） |
| `{{total:字段名}}` | 整列合计（放在表格区域下方） |

#### 表格内可用词条（行字段）

**收入结算（income）可用字段：**

| 词条 | 类型 | 说明 |
|------|------|------|
| `{{row.seq}}` | int | 序号 |
| `{{row.channel_name}}` | string | 渠道名称 |
| `{{row.game_id}}` | string | 游戏编号 |
| `{{row.game_name}}` | string | 游戏名称 |
| `{{row.project_code}}` | string | 项目编号 |
| `{{row.project_name}}` | string | 项目名称 |
| `{{row.month}}` | string | 月份 |
| `{{row.raw_revenue}}` | number | 原始流水 |
| `{{row.real_revenue}}` | number | 真实流水 |
| `{{row.vouchers}}` | number | 代金券 |
| `{{row.test}}` | number | 测试 |
| `{{row.welfare}}` | number | 福利币 |
| `{{row.bad_debt}}` | number | 坏账 |
| `{{row.total_deductions}}` | number | 扣除合计 |
| `{{row.split_rate}}` | number | 分成比例 |
| `{{row.channel_fee_rate}}` | number | 通道费率 |
| `{{row.tax_rate}}` | number | 税率 |
| `{{row.split_ratio}}` | number | 分成总比例 |
| `{{row.settlement_amount}}` | number | 结算金额 |

**付款结算（payment）额外字段：**

| 词条 | 类型 | 说明 |
|------|------|------|
| `{{row.publisher_name}}` | string | 研发商户名称 |
| `{{row.fixed_fee}}` | number | 固定费用（保底金），仅付款结算可用。参与结算公式：`结算金额 = 净流水 × 分成比例 × (1-通道费率) × (1-税率) + 固定费用`。可在结算查询界面付款结算 Tab 的「固定费用」列直接编辑，保存后立即重算结算金额 |

## 模板设计示例

### 示例1：基础对账单模板

```
         A                    B                     C                     D
 1 |  {{title}}                                                          |
 2 |                                                                      |
 3 |  对账单编号: {{bill_no}}          对账周期: {{period}}               |
 4 |  生成日期: {{date}}                                                  |
 5 |                                                                      |
 6 |  甲方: {{party_a.name}}                                             |
 7 |  地址: {{party_a.address}}                                          |
 8 |  电话: {{party_a.phone}}                                            |
 9 |  开户行: {{party_a.bank_name}}     账号: {{party_a.bank_account}}    |
10 |  税号: {{party_a.tax_id}}                                           |
11 |                                                                      |
12 |  乙方: {{party_b.name}}                                             |
13 |  ...（同上）                                                         |
14 |                                                                      |
15 | {{#table:rows}}                                                      |
16 |  序号  |  游戏名称  |  原始流水  |  扣除合计  |  分成比例  |  结算金额  |
17 | {{row.seq}} | {{row.game_name}} | {{row.raw_revenue}} | {{row.total_deductions}} | {{row.split_rate}} | {{row.settlement_amount}} |
18 | {{/table}}                                                            |
19 |                                                                      |
20 |  合计: 原始流水 {{total:raw_revenue}}    结算金额 {{total:settlement_amount}}  |
21 |                                                                      |
22 |  备注:                                                                |
23 |  {{notes}}                                                           |
```

**区域对应关系：**

| 区域 | 位置 | 词条 |
|------|------|------|
| 标题区 | 第1行 | `{{title}}` |
| 编号/周期区 | 第3-4行 | `{{bill_no}}`、`{{period}}`、`{{date}}` |
| 甲方信息区 | 第6-10行 | `{{party_a.*}}` |
| 乙方信息区 | 第12行起 | `{{party_b.*}}` |
| 数据表格区 | 第15-18行 | `{{#table:rows}}`...`{{/table}}`，内部 `{{row.*}}` |
| 汇总区 | 第20行 | `{{total:*}}` |
| 备注区 | 第23行 | `{{notes}}` |

## 表格区域规则

1. `{{#table:rows}}` 和 `{{/table}}` 之间的内容是表格区域
2. 区域内的**表头行**（不含 `{{row.*}}` 标记的行）保持静态，输出一次
3. 区域内**含 `{{row.*}}` 标记的行**是模板行，会根据数据行数复制多份
4. 模板行的样式（字体、颜色、边框、对齐、数字格式）会完全复制到每行输出
5. `{{total:字段名}}` 放在表格区域下方，会自动计算整列合计值

## 上传和使用

1. 在 Excel 中设计好模板，保存为 `.xlsx` 文件
2. 进入 **系统管理 → 对账模板**，点击"上传模板"
3. 填写模板名称，选择适用类型（收入结算/付款结算/通用）
4. 上传模板文件，确认上传
5. 进入 **结算查询** 页面，点击"导出对账单"
6. 选择甲方、乙方，再选择刚上传的模板
7. 点击"确认导出"下载生成的对账单
