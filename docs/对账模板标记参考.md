# 对账模板标记词条参考

## 基本信息 (系统自动填充)

| 标记 | 类型 | 说明 |
|------|------|------|
| `{{title}}` | 文本 | 对账单标题，自动识别"收入结算对账单"/"付款结算对账单" |
| `{{bill_no}}` | 文本 | 对账单编号，自动生成 `ST-YYYYMMDD-HHMM` |
| `{{period}}` | 文本 | 对账周期，如 `2026-03` 或 `2026-02 ~ 2026-03` |
| `{{date}}` | 文本 | 生成日期，如 `2026年05月15日` |
| `{{notes}}` | 文本 | 备注说明 |

## 表格区域

| 标记 | 说明 |
|------|------|
| `{{#table:rows}}` | 表格开始标记，放在表头上方 |
| `{{/table}}` | 表格结束标记，放在末行数据下方 |

### 数据行字段 (放在 `{{#table}}...{{/table}}` 之间)

| 标记 | 类型 | 说明 |
|------|------|------|
| `{{row.seq}}` | 数字 | 行号，从1自动递增 |
| `{{row.game_id}}` | 文本 | 游戏编号 |
| `{{row.game_name}}` | 文本 | 游戏名称 |
| `{{row.project_code}}` | 文本 | 项目编号 |
| `{{row.project_name}}` | 文本 | 项目名称 |
| `{{row.raw_revenue}}` | 金额 | 原始流水 |
| `{{row.real_revenue}}` | 金额 | 真实流水（原始流水 × 折扣率） |
| `{{row.vouchers}}` | 金额 | 代金券扣除 |
| `{{row.test}}` | 金额 | 测试扣除 |
| `{{row.welfare}}` | 金额 | 福利币扣除 |
| `{{row.bad_debt}}` | 金额 | 坏账扣除 |
| `{{row.total_deductions}}` | 金额 | 扣除合计 |
| `{{row.split_rate}}` | 比率 | 分成比例（小数） |
| `{{row.channel_fee_rate}}` | 比率 | 通道费率（小数） |
| `{{row.tax_rate}}` | 比率 | 税率（小数） |
| `{{row.split_ratio}}` | 比率 | 分成总比（公式计算值） |
| `{{row.settlement_amount}}` | 金额 | 结算金额 |
| `{{row.fixed_fee}}` | 金额 | 固定费用（仅付款结算模式） |
| `{{row.*}}` | 任意 | 支持任意自定义字段名 |

| `{{total:welfare}}` | 福利币合计 |

## 使用说明

1. 复制 `基础对账模板.xlsx` 的[对账单]Sheet 作为设计底稿
2. 用 `{{标记}}` 替换对应单元格内容
3. 表格区域必须用 `{{#table:rows}}` 和 `{{/table}}` 包裹
4. `{{#table}}` 与 `{{/table}}` 之间的静态行（如表头）保留不动
5. 带 `{{row.xxx}}` 的行作为模板行，会被重复复制到每行数据
6. 支持任意自定义 `{{row.xxx}}` 字段名，自动匹配传入数据
7. `{{total:xxx}}` 仅对数字类型字段求和
8. 设计完成后，通过 **基础数据 → 对账模板** 上传
9. 在 **结算查询 → 导出对账单** 时选择模板导出
